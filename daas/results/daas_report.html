What this MVP proves

The end-to-end distillation-as-a-service pipeline is real and working, not just a design doc:

  • Ingest: Captured 3 real Gemini Pro traces on FloorAI queries with full token accounting.
  • Distill: Pro extracted a structured WorkflowSpec for each (orchestrator prompt, workers, tools, domain rules, success criteria).
  • Generate: Emitted 3 runnable Python scaffolds using the Gemini SDK pattern (orchestrator + workers + tool stubs + formatter).
  • Replay: Executed each scaffold with gemini-3.1-flash-lite-preview on the same original queries.
  • Judge: Deterministic comparison of entity overlap, numeric parity, structural parity, and measured cost delta.

Caveats (honest)

  • MVP uses mock tools. Production path: user provides API endpoints or we infer from repo.
  • Only 3 queries, one domain (retail ops). V4 should scale to SWE-bench Verified (public, unit-test verified) per BENCHMARK_STRATEGY.md.
  • Replay invokes multiple LLM calls (orchestrator + N workers + formatter) — net cost depends on worker count. Prompt caching not yet applied.
  • Domain-agnostic generalization unproven beyond retail-ops.

Next

  • Apply Anthropic prompt caching (90% discount on repeated context) — closes the cost overhead V3 exposed.
  • Add SDK targets beyond Gemini: Claude Agent SDK, OpenAI Agents SDK, LangChain.
  • Wire the runtime as a visible page on attrition.sh with Chef-style live trace UI.
  • Support live connectors (user-provided APIs) alongside mocks.
  • Scale to SWE-bench Verified for ground-truth validation.
