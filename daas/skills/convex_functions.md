# convex_functions — Convex TypeScript actions + schema

## When to pick this lane

- User's backend is Convex (or they want to move to it).
- Agent needs durable reactive state + queued actions + realtime
  subscriptions.
- Scaffolds here run inside Convex's deployment, not as a standalone
  Python process.

## Reference

- Convex docs: `docs.convex.dev`
- Key primitives: `query` / `mutation` / `action` / `internalAction`,
  `crons`, `scheduler`

## schema.ts skeleton

```typescript
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  runs: defineTable({
    query: v.string(),
    status: v.union(v.literal("pending"), v.literal("running"),
                    v.literal("complete"), v.literal("error")),
    result_json: v.optional(v.string()),
    created_at: v.number(),
    completed_at: v.optional(v.number()),
  }).index("by_created", ["created_at"]),

  scratchpad: defineTable({
    run_id: v.id("runs"),
    section: v.string(),
    content: v.string(),
    updated_at: v.number(),
  }).index("by_run_section", ["run_id", "section"]),

  tool_events: defineTable({
    run_id: v.id("runs"),
    tool_name: v.string(),
    arguments_json: v.string(),
    result_json: v.string(),
    elapsed_ms: v.number(),
    ts: v.number(),
  }).index("by_run", ["run_id", "ts"]),
});
```

## orchestrator.ts skeleton

```typescript
import { internalAction } from "./_generated/server";
import { internal } from "./_generated/api";
import { v } from "convex/values";

export const runOrchestrator = internalAction({
  args: { run_id: v.id("runs"), query: v.string() },
  handler: async (ctx, args) => {
    // 1. PLAN — call LLM, parse plan JSON
    const plan = await callPlanLLM(args.query);
    // 2. DISPATCH — for each assignment, call the worker internalAction
    for (const a of plan) {
      await ctx.runAction(internal.domains.daas.orchestrator.runWorker, {
        run_id: args.run_id, assignment: a,
      });
    }
    // 3. COMPACT — read scratchpad, emit final
    const sections = await ctx.runQuery(
      internal.domains.daas.runs.listScratchpad, { run_id: args.run_id });
    const final = await callCompactLLM(sections);
    await ctx.runMutation(internal.domains.daas.runs.complete, {
      run_id: args.run_id, result: final,
    });
  },
});
```

## Key invariants

- `internalAction` for LLM calls (can make HTTP + run long).
- `mutation` for writes (reactive queries auto-refresh clients).
- `query` for reads (indexed).
- Provider API keys ONLY via `process.env.<VAR>`; Convex dashboard
  sets them per-deployment.
- Use `ctx.scheduler.runAfter(delay, fn, args)` for bounded retry.

## Known failure modes

- Forgetting to wait on `ctx.runAction` → orphaned worker calls.
- Writing to the same scratchpad section from parallel workers →
  races. Mitigation: scratchpad keyed by (run_id, section) with
  distinct section names per worker.

## Eval criteria

- `npx convex codegen` clean.
- `npx tsc --noEmit` clean.
- Smoke test invokes `runOrchestrator` with a canned plan stub,
  verifies scratchpad rows + final result.
